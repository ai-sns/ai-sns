# Web Sidebar 功能实现测试指南

## 已实现的功能

### 1. 搜索功能 ✅
- **LLM 搜索框**: 在 LLM Online 区域输入关键词可以实时过滤 LLM 服务
- **Tool 搜索框**: 在 AI Tools Online 区域输入关键词可以实时过滤工具
- **搜索范围**: 搜索会匹配名称、标题和描述字段
- **实时更新**: 输入时立即更新显示结果

### 2. 编辑功能 ✅
- **编辑按钮**: 在管理对话框中点击编辑按钮
- **编辑表单**: 包含以下字段
  - Name (必填)
  - Title
  - URL (必填)
  - Description
  - Icon Filename
- **保存更新**: 点击 Save 按钮保存更改
- **实时刷新**: 保存后自动刷新侧边栏和管理对话框

### 3. 删除功能 ✅
- **删除按钮**: 在管理对话框中点击删除按钮
- **确认提示**: 删除前会弹出确认对话框
- **软删除**: 使用 is_delete 标记，不会真正删除数据
- **实时刷新**: 删除后自动刷新侧边栏和管理对话框

### 4. 拖拽排序功能 ✅
- **拖拽手柄**: 每个项目左侧有拖拽图标
- **拖拽排序**: 可以通过拖拽改变项目顺序
- **位置更新**: 拖拽完成后自动更新数据库中的 position 字段
- **实时刷新**: 更新后自动刷新侧边栏显示

## 后端 API 端点

### 已添加的端点

1. **GET /api/system/web-mng**
   - 获取所有 web 管理项目
   - 按 position 排序

2. **POST /api/system/web-mng**
   - 创建新的 web 管理项目

3. **PUT /api/system/web-mng/{item_id}**
   - 更新指定项目
   - 支持更新所有字段（除了 id, web_id, create_time）

4. **DELETE /api/system/web-mng/{item_id}**
   - 软删除指定项目
   - 设置 is_delete = True

5. **PUT /api/system/web-mng/reorder**
   - 批量更新项目位置
   - 接收 [{id, position}] 数组

## 测试步骤

### 测试搜索功能
1. 启动应用并进入 Web 页面
2. 在 LLM Online 区域的搜索框输入关键词
3. 验证列表实时过滤
4. 清空搜索框，验证显示所有项目
5. 在 AI Tools Online 区域重复测试

### 测试编辑功能
1. 点击 "Manage" 按钮打开管理对话框
2. 点击某个项目的编辑按钮
3. 修改表单中的字段
4. 点击 "Save" 保存
5. 验证管理对话框和侧边栏都已更新

### 测试删除功能
1. 点击 "Manage" 按钮打开管理对话框
2. 点击某个项目的删除按钮
3. 在确认对话框中点击确认
4. 验证项目已从列表中移除
5. 验证侧边栏也已更新

### 测试拖拽排序功能
1. 点击 "Manage" 按钮打开管理对话框
2. 按住某个项目的拖拽手柄（左侧图标）
3. 拖动到新位置
4. 释放鼠标
5. 验证位置已更新
6. 关闭对话框，验证侧边栏中的顺序也已更新
7. 刷新页面，验证顺序保持不变（已保存到数据库）

## 注意事项

1. **数据库字段**: 确保 web_mng 表有 position 字段
2. **API 权限**: 确保后端 API 可以访问
3. **WebSocket**: 如果需要实时同步，可以考虑添加 WebSocket 通知
4. **图标上传**: 当前使用 filename 字段，未来可以添加图标上传功能
5. **验证**: 编辑表单有基本的必填验证

## 已知限制

1. 搜索功能是前端过滤，不支持后端分页搜索
2. 拖拽排序在移动端可能需要优化
3. 删除是软删除，需要定期清理数据库
4. 图标需要手动上传到 resource/images 目录
