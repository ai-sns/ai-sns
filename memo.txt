          const data = JSON.parse(event.data);
          handleWebSocketMessage(data);

  function handleWebSocketMessage(data) {
      switch (data.type) {

### 4. 地图页面与后端 api_server 的交互

在 [scripts/js/interact_python.js](file:///Users/xinhuang/my_projects/ai-sns/el/scripts/js/interact_python.js) 文件中，地图页面通过以下方式与后端 api_server 交互：

- [jsonrpcRequest](file://c:\dev\agi-ev\ai-sns-el\scripts\js\interact_python.js#L7-L40) 函数用于向 `/jsonrpc` 端点发起 JSON-RPC 2.0 请求

#### WebSocket 实时通信：
- 地图页面通过 WebSocket (`ws://localhost:8788/ws/{clientId}`) 与后端保持长连接
- 用于接收实时消息，如聊天消息、状态更新、警报、命令等
- 当收到消息时，根据消息类型执行相应的操作

### 5. 主要交互流程

1. **地图初始化**：
   - Electron 主进程启动后，通过 [createMapWindow()](file://c:\dev\agi-ev\ai-sns-el\electron\main.js#L97-L131) 创建地图窗口
   - 地图窗口加载 `http://localhost:8788/scripts/map.html`
   - 地图页面在 DOMContentLoaded 事件后连接 WebSocket 并加载地图配置

2. **配置加载**：
   - 地图页面通过 JSON-RPC 调用 [get_map_settings](file://c:\dev\agi-ev\ai-sns-el\api_server.py#L578-L631) 方法获取地图配置
   - 根据配置初始化地图中心点、用户位置、3D 模型等

3. **实时交互**：
   - 用户在地图上的操作（如移动、聊天）通过 WebSocket 发送到后端
   - 后端处理完成后，通过 WebSocket 将结果推回地图页面

4. **地图控制**：
   - Electron 主进程可以通过 IPC 发送命令到地图页面
   - 地图页面通过 [handle_command](file://c:\dev\agi-ev\ai-sns-el\scripts\js\interact_python.js#L117-L175) 函数处理来自主进程的命令
